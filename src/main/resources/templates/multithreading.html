Многопоточность

Многопоточность - возможность выполнения нескольких действий в одну единицу времени.
Сколько иммено зависит от колличества ядер.
Если многопоточная программа будет выполнятся на системе с одноядерным процессором значит задачи возложенные на разные потоки будут выполнятся не в одну еденицу в ремени, а поочередно (псевдо многопоточность).

Создание нового потока

  1. Наследоваться от Thread или реализовать Runnable
  2. Переопределить run(), написав в нем код которым будет заниматься новый поток
  3. Создать екземпляр класса который наследуется от Thread или создать новый Thread передав ему екземпляр класса реализующего Runnable
  4. Вызвать метод start (не run() !!!!!!!!) у екземпляра класса который наследуется от Thread или екземпляра Thread которому был передан екземпляр класса реализовывающего Runnable

Методы

start() - начать выполнение потока

run() - внутри этого метода пишем чем собственно новый поток будет заниматься

getName() - узнать имя потока

setName() - назвать поток

getPriority() - узнать приоритет потока

setPriority(int priority) - установить приоритет потока.
priority может быть от 1 до 10.
main поток по умолчанию 5.
Так оно в теории, а на практике что хочет то и делает т.к. наш priority не является единственным критерием по которому определяется приоритетность. А по какому еще определяется напишите в комментариях.

interrupt() - прервать выполнение потока

isAlive() - возвращает true, если поток активен

isInterrupted() - возвращает true, если поток был прерван

sleep(howLong) - спать(как долго в млс)

wait() - sleep пока не получит notify

notify() - разбудить

notifyAll() - разбудить всех кто wait()

join() - ждать завершения потока

Некоторые понятия

volatile - это запрет на запись переменной в кеш памяти дабы избежать ситуации когда значение переменной изменилось а в кеше не было поменяно.

Dead Lock (взаимная блокировка) - потоки ждут друг друга

Racing Condition (состояние гонки) - это недостаток, возникающий, когда время или порядок событий влияют на правильность программы

При работе с synchronized методом лочится доступ всех synchronized методов обЬекта для других потоков. Если synchronized метод еще и static лочится класс (т.к. метод пренадлежит классу а не обЬекту).
