Методы Класса Object

Класс Object - класс от которого не явно наследуются все классы в Java. Соответственно все классы в java из коробки имеют такие методы:

  • toString()
  • getClass()
  • equals()
  • hashCode()
  • + методы связанные с многопоточностью


toString() - возращает представление обЬекта в виде строки. Из коробки это представление вряд ли может быть хорошим описанием обЬекта по этому его переопределяют.

getClass() - позволяет узнать тип обЬекта (вернет "class " + Имя класса екземпляром которого является проверяемый обЬект).

equals() - сравнивает обЬекты по их ссылкам(Работает как == с обЬектами). Для сравнения обЬектов по их внутреннему содержимому метод нуждается в переопределении. Если переопределяется equals() должен быть переопределен и hashCode().

hashCode() - метод для оптимизации сравнения обЬектов. Работает быстрее тяжеловесного equals(). По умолчанию возращает цифровое представление адреса памяти обЬекта. Если был переопределен equals(), должен быть переопределен hashCode(). В вычислении нового хэша должны участвовать те же поля по которым у нас происходит сравнение в переопределенном equals(). Если хэши обЬектов не равны обЬекты гарантированно разные. Если равны имеет смысл проверить их по equals() чтобы убедится что они действительно равны, а не имеет место быть ситуация когда у обЬектов с различным содержимым просто совпали хэши(коллизия). Таким образом мы не используем тяжелый equals() каждый раз при сравнении. Если таким образом сравнивается 5 обЬектов может оно и не сильно заметно однако если их 5 миллионов я думаю польза hashCode() становится очевидна. Такой механизм сравнения обЬектов в java используют коллекции основанные на хэшах. Соответственно, если не соблюдать правила переопределения equals() и hashCode() упомянутые выше коллекции будут работать не корректно.
