Коллекции

Коллекции - это обЬекты для хранения обЬектов.

Найболее часто использующимися из них являются:

  • ArrayList
  • LinkedList
  • HashSet
  • TreeSet
  • HashMap
  • TreeMap

ArrayList - структура данных для хранения обЬектов упорядоченных по индексу на основе массива но в отличии от него изменяющая свою длинну. На самом деле под капотом у ArrayList обычный массив но он пересоздается при изменении его размеров.
Может хранить null значения.

LinkedList - структура данных для хранения обЬектов упорядоченных по индексу на основе двусвязного списка. Так же с LinkedList мы можем работать как с двунаправленной очередью.
Может хранить null значения.

HashSet - структура данных для хранения уникальных обЬектов на основе хеш таблиц. Ключем к обЬекту является hashCode этого же обЬекта.
Может хранить один Null. При добавлении второго Null будет перезаписан первый.

TreeSet - структура данных для хранения уникальных обЬектов упорядоченных по выбранному с помощью comparable или comparator параметру или параметрам на основе дерева.
Не может быть Null т.к. в таком случае не понятно как сортировать.

HashMap - структура данных для хранения пар обЬектов(ключ - значение).
Внутри HashMap устроен как массив связных списков(бакетов) каждый елемент которого имеет одну ссылку(на значение). В зависимости от хеш кода ключа добавляемой пары(ключ - значение) ключ попадает в один из связных списков(бакетов). Каждый елемент связного списка(ключ) имеет ссылку на значение. Таким образом дотигается быстрый доступ к елементам HashMap. Допустим у нас 10 бакетов по 10 елементов и чтобы найти нужный елемент нам нужно перебрать максимум 10 елементов массива и максимум 10 елементов бакета чтобы получить ссылку на нужное нам значение. Итого максимум 20 вместо 100.
Null может быть один ключ и сколько угодно значений. При добавлении второго Null ключа будет перезаписано значение первого.

TreeMap - структура данных для хранения пар обЬектов(ключ - значение) упорядоченных по выбранному с помощью comparable или comparator параметру или параметрам на основе дерева.
Сортировка осуществляется по ключам.
Не может быть ни одного Null ключа но может сколько угодно Null значений.

Каждая из приведенных выше коллекций имеет потокобезопасный аналог.

Некоторые понятия

Связной Список - структура данных в которой елементы имеют ссылку на следующий елемент.

Двусвязный Список - структура данных в которой елементы имеют ссылку на следующий и предидущий елементы. При такой организации хранения елементов удобно изменять их колличество (даже в середине списка) т.к. для этого придется просто поменять ссылки граничащих с новым или удаленным елементом.

Очередь - структура данных в которой елементы хранятся в порядке их добавления и мы имеем доступ только к тому который был добавлен первым. First in first out(FIFO).

Стэк - структура данных в которой елементы хранятся в порядке их добавления и мы имеем доступ только к тому который был добавлен последним. Last in first out(LIFO).

Двунаправленная Очередь - структура данных в которой мы имеем доступ только к первому и последнему елементу.

Хэш Таблица - структура данных в которой хранятся пары елементов (ключ - значение).

Дерево - структура данных при которой есть корневой елемент имеющий две ссылки (на больший и меньший по какому то признаку елемент). Каждый елемент так же имеет две ссылки (на больший и меньший). Елементы не имеющие ссылок называются листва.
Это позволяет хранить данные в сортированном по какому то свойству или свойствам виде и иметь быстрый доступ к елементам.

Выводы

Если нам неизвестно колличество обЬектов или оно будет постоянно менятся используем вместо массивов ArrayList.

Если мы в дальнейшем собираемся часто менять кол-во обЬектов в ArrayList используем LinkedList.

Если обЬекты должны быть уникальными HashSet.

Если уникальные обЬекты должны быть отсортированы по какому то признаку TreeSet.

Есть возможность хранить данные в виде ключ-значение. HashMap - для быстрой навигации. TreeMap - тоже быстро но медленне чем HashMap зато в отсортированном по выбранному нами параметру или параметрам виде.
